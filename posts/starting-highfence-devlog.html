highfence devlog를 다시 열며

왜 다시 여는가

예전에 GitHub Pages로 블로그를 한 번 만들려고 했던 흔적이 남아 있었다. Jekyll 테마를 가져와서 설정을 조금 만진 상태였는데, 정작 글을 쓰고 계속 갱신하는 흐름까지는 이어지지 못했다.

이번에는 조금 다르게 해보려고 한다.

어떤 글을 남길 것인가

거창한 기술 블로그보다는, 개발하면서 배운 것과 삽질한 것을 편하게 남기는 공간으로 쓰고 싶다. Unreal Engine, C++, 성능 최적화처럼 일을 하면서 자주 마주치는 주제도 쓰겠지만, 꼭 완성된 튜토리얼만 올리지는 않을 생각이다.

오늘 배운 것, 다시 보면 좋을 메모, 생각이 정리된 짧은 글, 그리고 가끔은 그냥 사는 이야기도 섞일 수 있다.

기준

기준은 단순하게 잡는다.

  • 다시 읽었을 때 쓸모가 있을 것
  • 공개해도 괜찮은 내용만 남길 것
  • 완벽하게 정리되지 않았다는 이유로 미루지 않을 것
